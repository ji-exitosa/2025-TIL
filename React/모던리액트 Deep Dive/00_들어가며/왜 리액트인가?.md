## 왜 리액트인가?
- 2011년 페이스북의 뉴스피드 페이지에서 처음 선보인 기술
- 전자정부 표준 프레임워크의 프론트엔드 개발 도구로 채택됨
	- 안정성과 유지보수성을 최우선으로 살피는 환경
- 개발자들이 리액트를 선호하는 이유
	- 명시적인 상태 변경
		- 단방향 바인딩만 지원 ↔ angular 양방향 바인딩
			- 코드 규모가 클 때, 상태변화 트레킹이 어려움
		- 간단함과 유연함 제공
	- JSX(JavaScript XML)
		- HTML에 자바스크립트 문법을 더한 JSX 사용
		- 자바스크립트 문법을 {}로 감싸야 한다
		- JSX는 리액트 외에도 사용할 수 있는 곳이 많다
